Modify `nanobot/agent/tools/gui.py` so the nanobot path consumes the shared runtime contract and enforces explicit acknowledgement before foreground fallback.

1. Extend `parameters` with two new booleans:
   - `require_background_isolation`
   - `acknowledge_background_fallback`
2. Update `execute()` signature to accept:
   - `require_background_isolation: bool = False`
   - `acknowledge_background_fallback: bool = False`
3. When `self._gui_config.background` is false, preserve current behavior.
4. When `self._gui_config.background` is true:
   - call `probe_isolated_background_support(sys_platform=sys.platform)`
   - resolve with `require_isolation=require_background_isolation` and `require_acknowledgement_for_fallback=True`
   - call `log_mode_resolution(logger, decision, owner="nanobot", task=task)` before `_run_task()`
5. Apply the result exactly like this:
   - `mode="blocked"`: return JSON immediately with `success=false`, `summary=decision.message`, `steps_taken=0`, `trace_path=null`, and `error=null`; do not add a structured `background_mode` object
   - `mode="fallback"` plus `acknowledge_background_fallback=False`: return JSON immediately with `success=false`, `summary=f"{decision.message} Re-run with acknowledge_background_fallback=true to continue in foreground."`, `steps_taken=0`, `trace_path=null`, and `error=null`; do not call `_run_task()` and do not add a structured `background_mode` object
   - `mode="fallback"` plus `acknowledge_background_fallback=True`: keep the raw backend and continue into `_run_task()`
   - `mode="isolated"`: create `XvfbDisplayManager` as today, wrap with `BackgroundDesktopBackend(active_backend, mgr, run_metadata={"owner": "nanobot", "task": task, "model": self._model})`, call `_run_task()`, and `await wrapped_backend.shutdown()` in `finally`
6. Remove the current inline non-Linux warning branch. Fallback/block visibility must come from the shared resolved-mode log line and the minimal JSON early-return summary only.
7. Do not use `async with active_backend:` after this change.
  </action>
