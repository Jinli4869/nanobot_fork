    platform: str

    parameters: tuple[str, ...] = ()

    success_count: int = 0
    failure_count: int = 0
    success_streak: int = 0
    failure_streak: int = 0

def compute_confidence(skill: Skill) -> float:
    total = skill.success_count + skill.failure_count
    return skill.success_count / total if total > 0 else 1.0

**Constructor changes:** Add these parameters to `GuiAgent.__init__()` (after `backend`):
- `trajectory_recorder: TrajectoryRecorder` — REQUIRED (no default). Import from `opengui.trajectory.recorder`.
- `memory_retriever: MemoryRetriever | None = None` — optional. Import from `opengui.memory.retrieval`.
- `skill_library: SkillLibrary | None = None` — optional. Import from `opengui.skills.library`.
- `skill_executor: SkillExecutor | None = None` — optional. Import from `opengui.skills.executor`. If skill_library is provided, executor should also be provided.
- `memory_top_k: int = 5` — max non-POLICY memory entries
- `skill_threshold: float = 0.6` — minimum score to attempt skill execution

Store all as `self._trajectory_recorder`, `self._memory_retriever`, `self._skill_library`, `self._skill_executor`, `self._memory_top_k`, `self._skill_threshold`.

**Rewrite `run()` method:** The current `run()` does retry logic wrapping `_run_once()`. Restructure it to:

1. **Start trajectory**: `self._trajectory_recorder.start(phase=ExecutionPhase.AGENT)` as the very first action.

2. **Retrieve memory** (once at start):
```python
memory_context = None
if self._memory_retriever is not None:
    results = await self._memory_retriever.search(task, top_k=self._memory_top_k + 10)  # over-fetch to allow POLICY carve-out
    # POLICY entries always included regardless of score
    from opengui.memory.types import MemoryType
    policies = [(e, s) for e, s in results if e.memory_type == MemoryType.POLICY]
    others = [(e, s) for e, s in results if e.memory_type != MemoryType.POLICY][:self._memory_top_k]
    memory_entries = policies + others
    memory_context = self._memory_retriever.format_context(memory_entries)
```

3. **Search skill library** (once at start):
```python
skill_match = None
if self._skill_library is not None:
    from opengui.skills.data import compute_confidence
    search_results = await self._skill_library.search(task, top_k=1)
    if search_results:
        skill, relevance = search_results[0]
        confidence = compute_confidence(skill)
        final_score = relevance * confidence
        if final_score >= self._skill_threshold:
            skill_match = (skill, final_score)
```

4. **Execute: skill path or free explore:**
- If `skill_match is not None` and `self._skill_executor is not None`:
  - Set phase to SKILL: `self._trajectory_recorder.set_phase(ExecutionPhase.SKILL, reason=f"Matched skill: {skill_match[0].name}")`
  - Execute skill via `self._skill_executor.execute(skill_match[0], ...)` — pass the current backend and any necessary context
  - If skill execution succeeds: fall back to `_run_once()` in AGENT phase to let agent call `done` or continue (per CONTEXT.md: "After skill completes all steps: fall back to ReAct mode with history")
  - If skill execution fails: set phase back to AGENT, fall back to `_run_once()` for free exploration
- If no skill match: proceed directly with `_run_once()` for free exploration

5. **Pass memory_context** to `_build_messages()`: Add `memory_context: str | None = None` parameter to `_build_messages()`. In `_build_messages()`, pass `memory_context=memory_context` to `build_system_prompt()`.

6. **Record trajectory steps** in `_run_once()`: After each step completes (after `_run_step()` returns), call:
```python
self._trajectory_recorder.record_step(
    action=result.action.to_dict() if hasattr(result.action, 'to_dict') else {"action_type": result.action.action_type},
    model_output=result.action_summary,
    screenshot_path=str(result.next_observation.screenshot_path) if result.next_observation and result.next_observation.screenshot_path else None,
)
```
This goes right after the existing `_write_trace()` call inside `_run_once()`.

7. **Finish trajectory** in `run()`: After the retry loop completes (or on success), call `self._trajectory_recorder.finish(success=result.success)`.

8. **Post-run skill maintenance** (at end of `run()`): If a skill was matched, perform the full maintenance pass per CONTEXT.md locked decision: (1) update confidence, (2) discard low-confidence, (3) check merge opportunities.
```python
if skill_match is not None and self._skill_library is not None:
    skill = skill_match[0]
    from dataclasses import replace
    from opengui.skills.data import compute_confidence

    # (1) Update confidence counters
    if result.success:
        updated = replace(skill, success_count=skill.success_count + 1,
                          success_streak=skill.success_streak + 1, failure_streak=0)
    else:
        updated = replace(skill, failure_count=skill.failure_count + 1,
                          failure_streak=skill.failure_streak + 1, success_streak=0)

    # (2) Discard rule: after min 5 attempts, if confidence < 0.3, remove
    new_conf = compute_confidence(updated)
    total_attempts = updated.success_count + updated.failure_count
    if total_attempts >= 5 and new_conf < 0.3:
        self._skill_library.remove(skill.skill_id)
    else:
        self._skill_library.update(skill.skill_id, updated)

        # (3) Check merge opportunities via add_or_merge
        # After confidence update, check if this skill should be merged
        # with a similar higher-confidence skill in the library
        await self._skill_library.add_or_merge(updated)
```
