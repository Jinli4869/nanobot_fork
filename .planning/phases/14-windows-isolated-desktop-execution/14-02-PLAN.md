Create `opengui/backends/windows_isolated.py` with a dedicated Windows isolated backend. Use these exact design rules:

1. Public class:
   - `class WindowsIsolatedBackend`
   - constructor signature: `__init__(self, inner: DeviceBackend, display_manager: Win32DesktopManager, run_metadata: dict[str, str] | None = None) -> None`
   - public methods: `preflight()`, `observe()`, `execute()`, `list_apps()`, `shutdown()`, `__aenter__()`, `__aexit__()`
   - `platform` property returns `self._inner.platform`

2. Lifecycle ownership:
   - acquire `background_runtime.GLOBAL_BACKGROUND_RUNTIME_COORDINATOR.lease(...)` in `preflight()`
   - `await self._display_manager.start()` before any worker/session setup
   - if the inner backend exposes `configure_target_display`, call it with the returned `DisplayInfo`
   - log one readiness line starting `windows isolated backend ready:` and include `owner=`, `task=`, `backend_name=windows_isolated_desktop`, `display_id=`, and `desktop_name=`

3. Worker-launch seam:
   - create `opengui/backends/windows_worker.py` and export `launch_windows_worker(*, desktop_name: str, width: int, height: int, control_path: str) -> subprocess.Popen | Any`
   - inside that helper, build `startupinfo = subprocess.STARTUPINFO()` and set `startupinfo.lpDesktop = f"WinSta0\\\\{desktop_name}"`
   - launch the child with a concrete module entry point such as `sys.executable -m opengui.backends.windows_worker --desktop-name ... --width ... --height ... --control-path ...`
   - the main process must not rely on late `SetThreadDesktop()` behavior for the long-lived event-loop thread

4. Target-desktop seam:
   - do not reuse `LocalDesktopBackend.observe()` or `LocalDesktopBackend.execute()` directly as the only desktop-switch mechanism
   - add patchable helpers `_start_worker_session(display_info)`, `_observe_via_worker(screenshot_path, timeout)`, `_execute_via_worker(action, timeout)`, `_stop_worker_session(cleanup_reason)`
   - `_start_worker_session(display_info)` must call `launch_windows_worker(desktop_name=self._display_manager.desktop_name, ...)`
   - `observe()` and `execute()` call those helpers after `_assert_started()`

5. Cleanup rules:
   - `shutdown(cleanup_reason: str = "normal")` is idempotent
   - on cancellation, use `cleanup_reason="cancelled"`
   - on startup failure after desktop creation, use `cleanup_reason="startup_failed"`
   - always clear `configure_target_display(None)` if it was set
   - always attempt `await self._display_manager.stop()` even if worker cleanup raises
   - log one cleanup line starting `windows isolated backend cleanup:` and include `cleanup_reason=`, `display_id=`, and `backend_name=windows_isolated_desktop`

6. Error handling:
   - re-raise `asyncio.CancelledError`
   - suppress secondary cleanup exceptions with `logger.exception(...)` after recording the primary failure path
   - preserve the shared runtime lease release even on failure

Keep this backend self-contained. Do not push Windows-specific attachment details into `BackgroundDesktopBackend` or `LocalDesktopBackend`.
