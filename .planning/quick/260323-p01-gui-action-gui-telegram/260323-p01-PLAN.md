# Quick Task 260323-p01 Plan

**Description:** 修复 GUI action 容错与 GUI 成功后 Telegram 完成消息延迟。
**Date:** 2026-03-23
**Mode:** quick

## Objective

让 OpenGUI 在遇到可恢复的 action payload 偏差时继续通过纠错重试完成任务，同时让 GUI 成功后的用户可见完成消息尽快回到 Telegram，不再被额外收尾逻辑或 typing 状态拖慢。

## Context

- `opengui/action.py` 是 GUI action 的唯一规范化边界，`opengui/agent.py` 的 `_run_step()` 负责把解析失败转成纠错重试。
- `nanobot/agent/tools/gui.py` 当前在 `GuiAgent.run()` 之后还会做轨迹总结和技能提取，这些后处理位于用户可见结果返回之前。
- `nanobot/agent/loop.py` 负责 `gui_task` 的最终回复路径，`nanobot/channels/telegram.py` 负责 progress/final outbound message 的 typing 生命周期。

## Tasks

1. **收紧并明确 GUI action 容错边界**
   - 修改 `opengui/action.py` 和 `opengui/agent.py`，把常见、可恢复的 GUI tool payload 偏差统一收敛在 parser/retry 边界处理：
     - 保持真正无效的坐标/字段校验继续失败。
     - 对可纠正的 action 输出，优先产出明确的 tool error 并消耗当前 step 的重试，而不是把整次 run 提前打成不可恢复失败。
     - 明确 terminal `done` 行为的归一化，避免模型轻微漂移让完成态丢失或误判。
   - 在 `tests/test_opengui.py` 增加回归用例，至少覆盖：
     - 一个“可恢复 action payload -> agent 重试后继续完成”的场景。
     - 一个“terminal done 归一化后得到稳定 success/failure 语义”的场景。

2. **把 GUI 成功后的用户可见返回从后处理里解耦**
   - 修改 `nanobot/agent/tools/gui.py`，让 `GuiAgent.run()` 的成功/失败结果先形成可立即返回的结构化 payload，再把 trajectory summary / skill extraction 变成不阻塞最终回复的 best-effort 收尾。
   - 只保留影响返回内容正确性的同步逻辑；不让 LLM 总结、技能抽取或 trace 附加工序阻塞 Telegram 完成消息。
   - 必要时在 `nanobot/agent/loop.py` 微调 `gui_task` 的 final reply 路径，但不要改变现有 message-tool suppress 语义。
   - 在 `tests/test_opengui_agent_loop.py` 或 `tests/test_message_tool_suppress.py` 增加回归，证明 GUI 工具一旦产出成功结果，外层 final reply 不再等待额外后处理才能生成。

3. **修正 Telegram typing/完成消息时机并补齐通道回归**
   - 修改 `nanobot/channels/telegram.py`，确保 GUI 任务的最终非 progress 回复进入发送路径时立即停止 typing，不因前序 progress 或发送细节继续维持“正在输入”状态。
   - 维持现有 progress message 行为，不要让普通 `_progress` 更新提前关闭 typing。
   - 新增或补充 Telegram 通道测试，优先放在 `tests/test_telegram_channel.py`；如果现有测试结构更合适，也可以扩展现有测试文件。至少覆盖：
     - progress outbound 不停止 typing。
     - GUI 完成后的最终 outbound 会立即停止 typing 并发送完成消息。

## Verification

- 运行 `uv run pytest tests/test_opengui.py tests/test_opengui_agent_loop.py tests/test_message_tool_suppress.py tests/test_telegram_channel.py -q`
- 如果 Telegram 通道测试最终落在其他文件，更新为对应的精确 `pytest` 路径并保持本 quick task 只验证这两类修复相关用例。
