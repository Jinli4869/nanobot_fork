<objective>
Harden the promoted-shortcut execution seam so extracted shortcuts are actually executable, then prove the full extraction-to-execution pipeline is stable on representative Android and macOS seams.

Purpose: SSTA-04 is not satisfied by promotion-only assertions. The promoted shortcuts must execute through the real `ShortcutExecutor` path. Today `ShortcutExtractor._to_skill_step()` preserves trace action fields in `SkillStep.parameters`, but `ShortcutExecutor._execute_step()` ignores those fields for non-fixed steps. This plan closes that seam and adds deterministic Android/macOS regression tests that exercise the real promotion pipeline.

Output: Updated non-fixed shortcut execution merge logic in `multi_layer_executor.py`; expanded Phase 31 regression tests that promote JSONL traces and execute the resulting shortcuts successfully on fake mobile and desktop backends.
</objective>

def _to_skill_step(self, step: dict[str, Any]) -> SkillStep:
    action = step.get("action", {})
    parameters = {
        key: value
        for key, value in action.items()
        if key != "action_type"
    }
    return SkillStep(
        action_type=str(action.get("action_type", "")),
        target=str(step.get("model_output", "")),
        parameters=parameters,
        ...
    )
