# Quick Task 260323-p01 Summary

## Goal

修复两类问题：
- GUI action 在模型输出轻微偏差时过早失败，导致同一任务出现多次 GUI attempt / 外层重试。
- GUI 任务已经成功后，Telegram 完成消息还要等待额外收尾 LLM 回合，返回过慢。

## What Changed

1. `opengui/action.py`
   - 在 `parse_action()` 前增加坐标 payload 规范化。
   - 允许常见可恢复格式：
     - `x: [417]` / `y: [129]` 这种单值列表坐标。
     - `x: [498, 441]` 且缺少 `y` 时，拆分成 `x=498, y=441`。
   - 保留真正非法 payload 的严格校验，避免把错误输入静默吞掉。

2. `nanobot/agent/tools/gui.py`
   - GUI tool 返回结构中增加 `model_summary`，保留 GUI 子代理已经得到的用户可见完成语义。

3. `nanobot/agent/loop.py`
   - 为成功的单次 `gui_task` 增加快速完成分支。
   - 当 GUI tool 已返回 `success=true` 时，优先用 `model_summary`/`summary` 直接生成最终回复，不再额外等待一轮 LLM 收尾。

## Why This Fix

- 第一个问题的根因在 action parser 边界，而不是 ADB/backend。
- 第二个问题的根因在主 agent 的 tool 后收尾路径，而不是 Telegram send 本身。
- 因此这次没有修改 `nanobot/channels/telegram.py`；typing 结束时机仍沿用现有 final outbound 机制，但 final outbound 会更早产生。
