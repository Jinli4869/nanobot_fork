# Milestone v1.1: Background Execution

**Status:** ✅ SHIPPED 2026-03-20
**Phases:** 9-11
**Total Plans:** 7

## Overview

v1.1 shipped background desktop execution for OpenGUI on Linux by introducing a virtual display abstraction, a lifecycle-safe backend wrapper, and first-class background-mode wiring in both the standalone CLI and nanobot integration path.

## Phases

### Phase 9: Virtual Display Protocol

**Goal**: The codebase has a well-defined, testable abstraction for virtual displays, with a no-op implementation for Android/tests and a working Xvfb implementation for Linux CI and production.
**Depends on**: Phase 8 (v1.0 complete)
**Plans**: 3 plans

Plans:

- [x] 09-00-PLAN.md — Wave 0 test stubs for all VDISP requirements
- [x] 09-01-PLAN.md — Protocol, DisplayInfo, NoOpDisplayManager, re-exports, and tests
- [x] 09-02-PLAN.md — XvfbDisplayManager with error handling, auto-increment, crash detection, and tests

**Details:**
- `VirtualDisplayManager` and `DisplayInfo` were formalized as reusable protocol-layer contracts.
- `NoOpDisplayManager` provides a fast, subprocess-free implementation for Android and test scenarios.
- `XvfbDisplayManager` launches Xvfb via `asyncio.subprocess`, captures stderr for diagnostics, detects pre-socket crashes, and auto-increments display numbers when lock files exist.
- The phase shipped with 21 focused tests covering protocol behavior and mocked Xvfb lifecycle paths.

### Phase 10: Background Backend Wrapper

**Goal**: Any `DeviceBackend` can be wrapped in `BackgroundDesktopBackend` to run GUI actions against a virtual display while setting `DISPLAY` and translating coordinates for non-zero-offset displays.
**Depends on**: Phase 9
**Plans**: 2 plans

Plans:

- [x] 10-01-PLAN.md — Test suite covering BGND-01 through BGND-04
- [x] 10-02-PLAN.md — Production `BackgroundDesktopBackend` with lifecycle guards, context manager, and idempotent shutdown

**Details:**
- `BackgroundDesktopBackend` was implemented as a thin decorator rather than a new concrete backend.
- The wrapper saves and restores `DISPLAY`, applies coordinate offsets only for absolute actions, and exposes an async context manager for safe lifecycle control.
- Shutdown is idempotent and suppresses display-manager teardown exceptions for best-effort cleanup.
- The phase shipped with 14 tests covering protocol conformance, guardrails, DISPLAY handling, offsets, and cleanup behavior.

### Phase 11: Integration & Tests

**Goal**: The `--background` flag is a first-class CLI option, nanobot `GuiConfig` supports background mode, and all new paths are verified by CI-safe tests with mocked subprocesses.
**Depends on**: Phase 10
**Plans**: 2 plans

Plans:

- [x] 11-01-PLAN.md — CLI background flags, wrapping logic, and CLI tests
- [x] 11-02-PLAN.md — Nanobot background config fields, execute() wrapping, and integration tests

**Details:**
- The standalone CLI gained `--background`, `--display-num`, `--width`, and `--height`, with validation that blocks incompatible ADB and dry-run combinations.
- Nanobot `GuiConfig` gained background display fields plus a validator enforcing `backend='local'` for background mode.
- Both CLI and nanobot wrap local desktop execution in `BackgroundDesktopBackend(XvfbDisplayManager(...))` on Linux and fall back to foreground execution with warnings on non-Linux platforms.
- The milestone closed with a full green regression run: `678 passed`.

## Milestone Summary

**Key Decisions:**

- Use a protocol layer (`VirtualDisplayManager`, `DisplayInfo`) so background execution stays decoupled from concrete display implementations.
- Use Xvfb through `asyncio.subprocess` instead of a Python binding to keep the runtime dependency surface small.
- Keep `BackgroundDesktopBackend` as a decorator wrapper rather than branching logic inside each backend implementation.
- Enforce Linux-only background execution at runtime while preserving a warning-based fallback for other platforms.

**Issues Resolved:**

- Added a production-ready Linux virtual display path without requiring a real Xvfb binary in CI.
- Closed the missing background-execution integration gap between `opengui.cli` and nanobot `GuiSubagentTool`.
- Added lifecycle-safe background execution semantics around `DISPLAY`, virtual display startup, and teardown.

**Issues Deferred:**

- macOS background execution via CGVirtualDisplay
- Windows background execution via CreateDesktop
- User-intervention detection and foreground handoff during background runs

**Technical Debt Incurred:**

- `11-02-SUMMARY.md` is missing `requirements-completed` metadata for `INTG-02` and `INTG-04`, leaving an audit-only traceability gap.
- `10-VALIDATION.md` and `11-VALIDATION.md` remain Nyquist-partial and should be normalized if strict workflow compliance is needed before the next milestone audit.
