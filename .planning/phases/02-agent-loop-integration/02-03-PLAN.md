**`NodeResult` dataclass** (not frozen â€” accumulates state):
```python
@dataclass
class NodeResult:
    success: bool
    steps_used: int = 0
    skills_attempted: dict[str, bool] = field(default_factory=dict)  # skill_id -> success
    error: str | None = None
```

**`TreeExecutor` dataclass:**
```python
@dataclass
class TreeExecutor:
    llm: LLMProvider
    backend: DeviceBackend
    recorder: TrajectoryRecorder
    skill_library: SkillLibrary | None = None
    skill_threshold: float = 0.6
    max_recovery_steps: int = 3
    max_steps: int = 15
    history_image_window: int = 4
```

Instance variable `_steps_remaining: int` initialized in `execute()` from `max_steps`.

**Core methods:**

1. `async def execute(self, node: SubgoalNode) -> NodeResult` â€” entry point. Sets `_steps_remaining = self.max_steps`. Delegates to `_execute_node()`.

2. `async def _execute_node(self, node: SubgoalNode) -> NodeResult` â€” pattern match on node type, dispatch to `_execute_and`, `_execute_or`, or `_execute_atom`.

3. `async def _execute_and(self, node: AndNode) -> NodeResult` â€” iterate children sequentially. If any child fails AND it's an AndNode, attempt replan (for now, just return failure â€” full replanning is future work). Accumulate `steps_used` and `skills_attempted` from children.

4. `async def _execute_or(self, node: OrNode) -> NodeResult` â€” iterate children until one succeeds. If all fail, return failure. Track steps across attempts.

5. `async def _execute_atom(self, node: AtomNode) -> NodeResult` â€” the key dispatch:
   - If `_steps_remaining <= 0`: return failure with "step budget exhausted"
   - Call `_find_skill_match(node.instruction)` to get best skill + final score
   - If match found (final_score >= threshold): call `_execute_with_skill(skill, node.instruction)`
   - If no match: call `_free_explore(node.instruction)`

6. `async def _find_skill_match(self, instruction: str) -> tuple[Skill, float] | None`:
   - If `self.skill_library is None`: return None
   - Call `await self.skill_library.search(instruction, top_k=1)`
   - If no results: return None
   - Compute `confidence = success_count / (success_count + failure_count)` (1.0 if total == 0)
   - `final_score = search_score * confidence`
   - Return `(skill, final_score)` if `final_score >= self.skill_threshold`, else None

7. `async def _execute_with_skill(self, skill: Skill, instruction: str) -> NodeResult`:
   - Set `recorder.set_phase(ExecutionPhase.SKILL)`
   - Create `SkillExecutor` with `self.llm`, `self.backend`, recorder reference
   - Call `await executor.execute(skill, params={})` (parameter pre-filling from instruction is at Claude's discretion â€” recommend extracting params via LLM in a separate step or keeping empty for v1)
   - Record steps used, track skill outcome in NodeResult.skills_attempted
   - Decrement `_steps_remaining` by steps used
   - If skill execution fails (valid_state failure): call `_recovery_agent(failed_valid_state_description)`
   - If recovery succeeds: continue remaining skill steps
   - If recovery fails: fall back to `_free_explore(instruction)` with remaining budget
   - After skill completes all steps: fall back to ReAct mode via `_free_explore(instruction)` with subgoal instruction so agent can call `done` to signal completion
   - Set `recorder.set_phase(ExecutionPhase.AGENT)` when entering free-explore after skill

8. `async def _free_explore(self, instruction: str, *, max_steps: int | None = None) -> NodeResult`:
   - This is the refactored `_run_once()` step loop from GuiAgent
   - Use `max_steps or self._steps_remaining` as the step limit
   - Observe screen, build messages with instruction as the task, call LLM, parse action, execute on backend
   - Record each step via `recorder.record_step()`
   - Decrement `_steps_remaining` per step
   - Stop when LLM calls `done` action or step budget exhausted
   - Return NodeResult with success based on whether `done` was called with `status="success"`

9. `async def _recovery_agent(self, valid_state_desc: str) -> bool`:
   - Set `recorder.set_phase(ExecutionPhase.RECOVERY)`
   - Call `_free_explore(valid_state_desc, max_steps=self.max_recovery_steps)` with the valid_state description as instruction
   - Recovery steps do NOT decrement `_steps_remaining` (separate budget)
   - Return True if recovery succeeded (agent called `done` with success), False otherwise

**Confidence helper** (module-level function):
```python
def _confidence(skill: Skill) -> float:
    total = skill.success_count + skill.failure_count
    if total == 0:
        return 1.0
    return skill.success_count / total
```

**New tree-based flow in `run()` (when trajectory_recorder is set):**

1. **Start trajectory**: `self.trajectory_recorder.start()`
2. **Retrieve memory once** (if `self.memory_retriever` is not None):
   - Get all entries from memory store (the caller is responsible for providing indexed retriever)
   - `results = await self.memory_retriever.search(task, top_k=self.memory_top_k)`
   - POLICY pinning: also retrieve all POLICY entries and ensure they're included
   - `memory_context = self.memory_retriever.format_context(results)`
3. **Observe initial screen**: `obs = await self.backend.observe(run_dir=run_dir)`
4. **Plan**: `tree = await self._planner.plan(task, obs.screenshot_path, memory_context)`
5. **Create TreeExecutor** with `self.llm`, `self.backend`, `self.trajectory_recorder`, `self.skill_library`, `self.skill_threshold`, `self.max_steps`, `self.history_image_window`
6. **Execute tree**: `node_result = await tree_executor.execute(tree)`
7. **Post-run maintenance**: `await self._post_run_maintenance(node_result)`
8. **Finish trajectory**: `self.trajectory_recorder.finish(success=node_result.success)`
9. **Return AgentResult**

**`_post_run_maintenance(self, node_result: NodeResult)` method:**
- For each `(skill_id, success)` in `node_result.skills_attempted`:
  - Get skill from library
  - Use `dataclasses.replace()` to update confidence fields:
    - If success: `success_count += 1`, `success_streak += 1`, `failure_streak = 0`
    - If failure: `failure_count += 1`, `failure_streak += 1`, `success_streak = 0`
  - Discard rule: if `total >= 5` and `_confidence(updated) < 0.3`, remove from library
  - Otherwise upsert the updated skill
- Save changes to library

**Keep `_run_once()` intact** for backward compatibility. P0 regression tests still call `GuiAgent(llm, backend)` without trajectory_recorder and expect the flat step loop.
