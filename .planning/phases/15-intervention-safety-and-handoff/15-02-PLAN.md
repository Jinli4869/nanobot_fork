  <action>
Implement the Phase 15 agent seam with these exact target states:

1. In `opengui/interfaces.py`, add:
   - `@dataclass(frozen=True) class InterventionRequest` with fields:
     - `task: str`
     - `reason: str`
     - `step_index: int`
     - `platform: str`
     - `foreground_app: str | None`
     - `target: dict[str, typing.Any]`
   - `@dataclass(frozen=True) class InterventionResolution` with fields:
     - `resume_confirmed: bool`
     - `note: str | None = None`
   - `class InterventionHandler(typing.Protocol)` with:
     - `async def request_intervention(self, request: InterventionRequest) -> InterventionResolution: ...`

2. In `opengui/agent.py`:
   - add an optional `intervention_handler: InterventionHandler | None = None` constructor argument
   - when `_run_step()` parses `action_type="request_intervention"`, do not call `backend.execute()` and do not call `backend.observe()`
   - return a structured step result that includes the intervention reason and skips the normal execution snapshot path
   - in `_run_once()`, when an intervention result is returned:
     - create an `InterventionRequest`
     - call the handler if present; if no handler is present, fail closed with a clear error summary
     - record `intervention_requested`, `intervention_resumed`, and `intervention_cancelled` events via `_log_attempt_event(...)` and `TrajectoryRecorder.record_event(...)`
     - after `resume_confirmed=True`, capture one new screenshot into the next step path and continue from that observation
     - if `resume_confirmed=False`, return a non-success `AgentResult` whose error mentions `intervention_cancelled`

3. In `opengui/trajectory/recorder.py` and the logging helpers:
   - preserve the generic `record_event()` API
   - ensure the intervention payloads passed into it are already scrubbed
   - extend `GuiAgent._scrub_for_log()` so it redacts:
     - `text` on `action_type="input_text"`
     - intervention `reason`
     - keys containing `password`, `secret`, `token`, `otp`, or `credential`
   - use exact placeholder strings:
     - `<redacted:input_text>`
     - `<redacted:intervention_reason>`

Do not wire CLI or nanobot in this plan; that comes next.
