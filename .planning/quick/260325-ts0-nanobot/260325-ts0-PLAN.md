<objective>
Wire SkillExecutor into nanobot's GuiSubagentTool so that skill execution is available when the GUI agent finds a matching skill.

Purpose: Currently, when nanobot calls GuiAgent, skill_library is passed (skills can be matched) but skill_executor is NOT passed, so the condition `skill_match is not None and self._skill_executor is not None` at opengui/agent.py:221 always skips execution. This change makes skill execution opt-in via config.

Output: Updated GuiConfig with `enable_skill_execution` field, wired SkillExecutor in GuiSubagentTool._run_task, unit tests.
</objective>

   - Before the `GuiAgent(...)` constructor call, add:

     skill_executor = None
     if self._gui_config.enable_skill_execution:
         from opengui.skills.executor import LLMStateValidator, SkillExecutor
         skill_executor = SkillExecutor(
             backend=active_backend,
             state_validator=LLMStateValidator(self._llm_adapter),
         )

   - Pass `skill_executor=skill_executor` to the `GuiAgent(...)` constructor call.
   - The lazy import keeps opengui.skills.executor out of the critical import path.

3. In `tests/test_gui_skill_executor_wiring.py`, write unit tests:
