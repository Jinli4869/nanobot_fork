# Quick Task 260324-k9r Plan

**Description:** 修复技能提取：统一 `gui_skills` 存储根目录并规范化提取出的 app 标识，恢复 `SkillLibrary` 的合并与管理能力。  
**Date:** 2026-03-24  
**Mode:** quick

## Objective

让 GUI skill 的持久化路径和 app 分桶规则回到单一规范：同一平台、同一应用的技能必须稳定写入同一个 `gui_skills` 桶，提取阶段产生的 app 标识也必须在入库前被规范化，从而让 `SkillLibrary.add_or_merge()`、`list_all()` 和后续管理能力重新命中同一组技能。

## Constraints

- 只修复 `gui_skills` 根目录与 app 标识规范化，不扩展新的 skill schema。
- 保持现有 `SkillLibrary` 的 `{platform}/{app}/skills.json` 存储模型，但消除调用方再次预分层导致的路径漂移。
- 规范化必须是确定性的，不能依赖运行时 LLM 输出的大小写、包名别名或自然语言描述。

## Tasks

1. 梳理并统一 GUI skill store 根目录约定
   - 明确 `GuiSubagentTool` 传给 `SkillLibrary` 的 `store_dir` 应始终是统一根目录 `workspace/gui_skills`，不要把 `platform` 预先拼进调用方。
   - 如有必要，补一个共享 helper 或常量，让写入、读取、测试都围绕同一根目录约定，避免 `SkillLibrary` 内部再拼 `platform/app` 时形成双层平台目录。
   - 更新受影响测试断言，确保工具层与库层对根目录语义一致。

2. 为提取出的 skill app 标识建立规范化入口
   - 在 `SkillExtractor` 解析响应或 `SkillLibrary` 入库边界增加统一的 app normalization，覆盖大小写、空白、常见自然语言 app 名称/包名混用等情形。
   - 让冲突检测、持久化分桶、列举过滤都基于规范化后的 `skill.app`，保证同一应用不会因为 `"Settings"`、`"settings"`、`"com.android.settings"` 等变体被拆成多个桶。
   - 保持平台维度不变，避免把不同平台的同名 app 错合并。

3. 补回回归测试，锁定合并与管理能力
   - 为 `SkillLibrary` 增加回归用例，验证同 app 的不同标识写入后会落到同一 bucket，并能触发 `add_or_merge()` 的冲突判定。
   - 为 `GuiSubagentTool` 增加端到端断言，验证自动提取后的 skill 被写入统一 `gui_skills` 根目录下的规范化 app 路径。
   - 必要时补充 load/reload 场景，证明重启后 `list_all()`/过滤查询仍能看到合并后的同一组技能。
