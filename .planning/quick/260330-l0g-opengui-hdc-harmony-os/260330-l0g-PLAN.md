  <name>Task 1: Create HdcBackend for HarmonyOS device automation</name>
  <files>opengui/backends/hdc.py</files>
  <action>
Create `opengui/backends/hdc.py` following the AdbBackend pattern (async subprocess via `asyncio.create_subprocess_exec`). The HdcBackend class must implement the full DeviceBackend protocol.

Key implementation details from the HDC reference:

**Constructor:** `__init__(self, serial: str | None = None, hdc_path: str = "hdc")`. Store `_serial`, `_hdc`, default screen 720x1280.

**_build_cmd():** Build command list. If serial is set, use `[hdc, "-t", serial, ...]` otherwise `[hdc, ...]`. All subcommands go after the target flag.

**_run():** Same async pattern as AdbBackend — `asyncio.create_subprocess_exec`, `asyncio.wait_for` with timeout, decode stdout/stderr, raise `HdcError` on non-zero exit.

**preflight():** Run `hdc list targets`. Parse output lines for device IDs. If serial specified, verify it appears. If no serial, verify at least one device is listed. Raise HdcError with clear message on failure.

**list_apps():** HarmonyOS does not have a simple package list command like Android. Return a static list of common HarmonyOS system bundles as fallback:
  - `com.huawei.hmos.settings`, `com.huawei.hmos.camera`, `com.huawei.hmos.photos`, `com.huawei.hmos.contacts`, `com.huawei.hmos.mms`, `com.huawei.hmos.calendar`, `com.huawei.hmos.calculator`, `com.huawei.hmos.clock`, `com.huawei.hmos.filemanager`, `com.huawei.hmos.browser`

**observe():**
  1. Screenshot: Try `hdc shell screenshot /data/local/tmp/__opengui_cap.jpeg` first. If that fails (non-zero exit), fallback to `hdc shell snapshot_display -f /data/local/tmp/__opengui_cap.jpeg`.
  2. Pull: `hdc file recv /data/local/tmp/__opengui_cap.jpeg {local_path_as_jpeg}`.
  3. Convert JPEG to PNG: HarmonyOS screenshots are JPEG. Use `PIL.Image.open(jpeg_path).save(screenshot_path, format="PNG")` inside `asyncio.to_thread`. Import PIL lazily with helpful error message if missing. Save to the requested screenshot_path (which expects PNG for consistency with other backends).
  4. Screen size: Parse `hdc shell hidumper -s RenderService` output for resolution pattern like `(\d+)\s*x\s*(\d+)`. Default 720x1280 on failure.
  5. Foreground app: Run `hdc shell aa dump -l`. Parse for mission with `state #FOREGROUND` and extract bundle name. Use regex to find `bundle_name.*?#string\[(.*?)\]` or similar pattern from the aa dump output. Default to "unknown".
  6. Run screen size and foreground app queries in parallel via `asyncio.gather`.

**execute():** Dispatch action types using `hdc shell uitest uiInput` commands:
  - `tap`: `hdc shell uitest uiInput click X Y`
  - `double_tap`: `hdc shell uitest uiInput doubleClick X Y`
  - `long_press`: `hdc shell uitest uiInput longClick X Y`
  - `swipe`/`drag`: `hdc shell uitest uiInput swipe X1 Y1 X2 Y2 speed` (speed in px/s, convert duration_ms: speed = distance / (duration_ms/1000), default 2000)
  - `input_text`: `hdc shell uitest uiInput inputText X Y "text"` — Note: HDC text input requires coordinates. Use screen center (width//2, height//2) as default tap point, then input text. If action has coordinates, use those.
  - `hotkey`: Map keys — Back: `hdc shell uitest uiInput keyEvent Back`, Home: `hdc shell uitest uiInput keyEvent Home`, Enter: `hdc shell uitest uiInput keyEvent 2054`, Delete: `hdc shell uitest uiInput keyEvent 2055`
  - `scroll`: Calculate swipe endpoints from direction + pixels, use `uitest uiInput swipe`
  - `wait`: `asyncio.sleep`
  - `back`: `hdc shell uitest uiInput keyEvent Back`
  - `home`: `hdc shell uitest uiInput keyEvent Home`
  - `done`: pass (terminal)
  - `open_app`: `hdc shell aa start -b {bundle} -a {ability}`. For the ability, derive from bundle: use "MainAbility" as default ability name (standard HarmonyOS convention). If action.text contains "/" separator, split into bundle/ability.
  - `close_app`: `hdc shell aa force-stop {bundle}`

**Coordinate helpers:** Same pattern as AdbBackend — `_resolve_x`, `_resolve_y`, `_resolve_point`, `_resolve_second_point` using `resolve_coordinate` from `opengui.action`.

**platform property:** Return `"harmonyos"`.

**HdcError exception:** Same pattern as AdbError with `message`, `returncode`, `stderr` fields.

**Key mapping dict:** `_HDC_KEYCODE_MAP` mapping lowercase key names to HDC keyEvent values:
  - home -> "Home", back -> "Back", enter -> "2054", return -> "2054", delete -> "2055", backspace -> "2055", volumeup -> "2072", volumedown -> "2073", power -> "2050"

**JPEG temp path constant:** `_DEVICE_SCREENSHOT_PATH = "/data/local/tmp/__opengui_cap.jpeg"`
