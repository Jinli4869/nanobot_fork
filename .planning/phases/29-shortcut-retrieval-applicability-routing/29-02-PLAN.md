<objective>
Add screen-aware applicability evaluation that gates shortcut execution on live
precondition checks, and wire the real ConditionEvaluator into the nanobot host path.

Purpose: SUSE-02 requires that runtime selection executes a shortcut only when current
screen evidence satisfies its applicability checks. The current code executes any skill
above the score threshold without checking preconditions. This plan adds an explicit
applicability gate between retrieval (Plan 01) and execution, producing a structured
decision (run/skip/fallback) with full traceability.

Output: Modified `GuiAgent.run()` with applicability evaluation using the first
observation's screenshot, nanobot wiring of `ShortcutApplicabilityRouter`, and full
SUSE-02 regression tests.

3. Add `_evaluate_shortcut_applicability` async method:
   ```python
   async def _evaluate_shortcut_applicability(
       self,
       candidates: list,
       *,
       screenshot_path: Path | None,
       task: str,
   ) -> ApplicabilityDecision:
       """Evaluate shortcut candidates against live screen and return decision."""
       if not candidates:
           decision = ApplicabilityDecision(outcome="fallback", reason="no_candidates")
           self._trajectory_recorder.record_event(
               "shortcut_applicability",
               outcome=decision.outcome,
               shortcut_id=None,
               reason=decision.reason,
           )
           return decision

       if self._shortcut_applicability_router is None or screenshot_path is None:
           # No router or no screenshot: use first candidate by score as best-effort
           best = candidates[0]
           decision = ApplicabilityDecision(
               outcome="run",
               shortcut_id=best.skill.skill_id,
               reason="no_applicability_router",
               score=best.score,
           )
           self._trajectory_recorder.record_event(
               "shortcut_applicability",
               outcome=decision.outcome,
               shortcut_id=decision.shortcut_id,
               reason=decision.reason,
               score=decision.score,
           )
           return decision

       # Evaluate candidates in score order, pick first passing
       for result in candidates:
           decision = await self._shortcut_applicability_router.evaluate(
               result.skill, Path(screenshot_path),
           )
           if decision.outcome == "run":
               decision = ApplicabilityDecision(
                   outcome="run",
                   shortcut_id=result.skill.skill_id,
                   reason=decision.reason,
                   score=result.score,
               )
               self._trajectory_recorder.record_event(
                   "shortcut_applicability",
                   outcome=decision.outcome,
                   shortcut_id=decision.shortcut_id,
                   reason=decision.reason,
                   score=decision.score,
               )
               return decision

       # All candidates failed applicability
       last_decision = decision  # from the loop
       fallback = ApplicabilityDecision(
           outcome="fallback",
           shortcut_id=None,
           reason=f"all_candidates_failed:{last_decision.reason}",
           failed_condition=last_decision.failed_condition,
       )
       self._trajectory_recorder.record_event(
           "shortcut_applicability",
           outcome=fallback.outcome,
           shortcut_id=None,
           reason=fallback.reason,
           failed_condition=(
               {"kind": fallback.failed_condition.kind, "value": fallback.failed_condition.value}
               if fallback.failed_condition else None
           ),
       )
       return fallback
   ```

4. Restructure `run()` to integrate applicability evaluation. The key change is moving shortcut execution inside the first retry loop iteration so the initial observation screenshot is available for applicability checking:

   In the retry loop (step 5), BEFORE calling `_run_once()` on the FIRST attempt only:
   - After `run_dir = self._make_run_dir(task, attempt)` and before `_run_once()` call:
   - If `attempt == 0 and shortcut_candidates`:
     - Take a pre-loop observation: `pre_obs = await self.backend.observe(run_dir / "screenshots" / "pre_shortcut_check.png", timeout=self.step_timeout)`
     - Call `applicability_decision = await self._evaluate_shortcut_applicability(shortcut_candidates, screenshot_path=pre_obs.screenshot_path, task=task)`
     - If `applicability_decision.outcome == "run"`:
       - Find the matched SkillSearchResult from shortcut_candidates where `skill.skill_id == applicability_decision.shortcut_id`
       - Set `matched_skill = found_result.skill`, `final_score = applicability_decision.score`
       - Inject memory context: `memory_context = await self._inject_skill_memory_context(matched_skill, memory_context)`
       - Execute skill if executor available (reuse existing step 4 pattern)
     - If `applicability_decision.outcome != "run"`: proceed normally (matched_skill stays from legacy _search_skill if any)

   - On retry (attempt > 0), if shortcut was attempted on first try and failed:
     - Clear `matched_skill = None`, `skill_context = None` so retries use free exploration

   The existing `_search_skill()` / step 3 legacy path is preserved. When `shortcut_candidates` is non-empty and the applicability router returns "run", the applicability-selected shortcut takes priority over the legacy `_search_skill` match.
