<objective>
Fix planner route selection so GUI subtasks use the correct route_id (gui.adb vs gui.desktop) based on the active backend.

Purpose: When AgentLoop triggers the planner for complex tasks, the planner currently always emits gui.desktop for GUI subtasks because it has no awareness of which backend is active. This causes all GUI subtasks to be routed incorrectly when the target is an Android phone via ADB.

Output: PlanningContext carries active_gui_route, CapabilityCatalogBuilder emits the correct gui route_id, planner prompt includes an explicit routing directive, and the router dispatches gui.adb atoms correctly.
</objective>
