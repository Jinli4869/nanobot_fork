```python
def update(self, skill_id: str, updated_skill: Skill) -> bool:
    """Replace a skill by ID. Returns True if skill existed and was replaced, False if not found.

    The updated_skill MUST have the same skill_id. This is used for
    post-run confidence updates (success_count, failure_count, streaks).
    """
    if skill_id not in self._skills:
        return False
    # Remove old entry and upsert the updated one
    self._remove_internal(skill_id)
    self._upsert(updated_skill)
    return True
```

Note: `_upsert()` and `_remove_internal()` are existing private methods that handle the internal dict + search index updates. The `update()` method does a remove-then-add to ensure the search index is refreshed with any changed fields.

Rewrite MemoryStore internals to use per-type markdown files instead of a single `memory.json`. The public API (add, remove, get, list_all, save, load, count) MUST remain identical.

**File layout:** Instead of `{store_dir}/memory.json`, create up to 4 files:
- `{store_dir}/os_guide.md`
- `{store_dir}/app_guide.md`
- `{store_dir}/icon_guide.md`
- `{store_dir}/policy.md`

**Markdown format per entry (H2 section):**
```markdown
## {content_first_line_or_truncated_summary}
id: {entry_id}
type: {memory_type.value}
platform: {platform}
app: {app_or_empty}
tags: {comma_separated_tags}
created_at: {float_timestamp}
access_count: {int}

{content}
```

Each `## ` heading marks an entry boundary. Metadata lines follow immediately (lines matching `key: value` before the first blank line). Content is everything after the blank line separator until the next `## ` or EOF.

**Implementation details:**
- `_type_to_filename(memory_type: MemoryType) -> str` helper maps type to filename
- `save()` rewrites ALL markdown files (not incremental) using atomic write (tempfile + rename, same pattern as current JSON)
- `load()` parses all 4 markdown files on init, using `re.split(r'^## ', content, flags=re.MULTILINE)` to chunk by H2
- For each chunk: parse metadata lines (key: value before first blank line), extract entry_id from `id:` line, reconstruct MemoryEntry via `from_dict()`
- Entry ID is stored explicitly in the `id:` metadata line (NOT derived from heading text) to avoid roundtrip issues
- If a markdown file does not exist, skip it (empty type)
- If the old `memory.json` exists, auto-migrate on load: read JSON, populate `_entries`, call `save()` to write markdown, then delete `memory.json`
