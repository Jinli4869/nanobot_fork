Concrete missing-shortcut fallback rule for the current schema:
```text
- `ShortcutRefNode` has no embedded fallback field in Phase 24, so the locked "same node" fallback must be derived structurally.
- The same-node fallback block for a `ShortcutRefNode` is the maximal contiguous run of `SkillStep` siblings immediately after that node in the same tuple.
- If `shortcut_resolver(node.shortcut_id)` returns a `ShortcutSkill`, execute it with `node.param_bindings` and skip that contiguous fallback block.
- If resolution returns `None` and the contiguous fallback block has one or more `SkillStep` nodes, execute those inline steps in order through the same shared step runner used by `ShortcutExecutor`.
- If resolution returns `None` and the next sibling is not a `SkillStep`, return `MissingShortcutReport` immediately.
- `BranchNode` siblings are never part of a fallback block.
```
