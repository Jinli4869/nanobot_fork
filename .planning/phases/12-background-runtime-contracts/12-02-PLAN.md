  <action>
Modify `opengui/cli.py` to use the Phase 12 runtime contract directly.

1. Extend `parse_args()` with a new flag:
   - `--require-isolation`
   - help text: `Block instead of falling back when isolated background execution is unavailable`
2. Add parser validation:
   - `--require-isolation` must error unless `--background` is also present
3. In `run_cli()`:
   - keep `task`, `config`, `backend`, and `provider` setup as-is
   - when `args.background` is false, preserve current foreground behavior
   - when `args.background` is true, call `probe_isolated_background_support(sys_platform=sys.platform)`
   - resolve with `require_isolation=args.require_isolation` and `require_acknowledgement_for_fallback=False`
   - call `log_mode_resolution(logger, decision, owner="cli", task=task)` before any call to `_execute_agent()`
4. Apply the resolved mode exactly like this:
   - `mode="blocked"`: raise `RuntimeError(decision.message)` before agent creation
   - `mode="fallback"`: keep the raw backend and call `_execute_agent()` without wrapping
   - `mode="isolated"`: instantiate `XvfbDisplayManager` exactly as today, wrap with `BackgroundDesktopBackend(backend, mgr, run_metadata={"owner": "cli", "task": task})`, then call `_execute_agent()` inside `try/finally` and `await wrapped_backend.shutdown()` in the `finally`
5. Remove the current non-Linux inline warning path from `run_cli()`. The fallback/block message must come from the shared resolved-mode log line and, for blocked runs, the raised `RuntimeError(decision.message)` only.
6. Do not use `async with backend:` in the CLI path after this change. The wrapper must rely on `GuiAgent.run()` calling `preflight()`, and the CLI must own `shutdown()` explicitly in `finally`.
  </action>
