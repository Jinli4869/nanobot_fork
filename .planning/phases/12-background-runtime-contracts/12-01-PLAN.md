3. Add frozen dataclasses exactly named `IsolationProbeResult` and `ResolvedRunMode` with the fields shown in the interface block.
4. Implement `normalize_host_platform()` with this exact mapping:
   - `"linux"` and `"linux2"` -> `"linux"`
   - `"darwin"` -> `"macos"`
   - `"win32"` and `"cygwin"` -> `"windows"`
   - anything else -> `"unknown"`
5. Implement `probe_isolated_background_support()` as a side-effect-free check:
   - on normalized `"linux"`, use `shutil.which(xvfb_binary)`
   - if found, return `reason_code="xvfb_available"`, `supported=True`, `retryable=False`, `backend_name="xvfb"`
   - if missing, return `reason_code="xvfb_missing"`, `supported=False`, `retryable=True`, `backend_name="xvfb"`
   - on all non-Linux hosts, return `reason_code="platform_unsupported"`, `supported=False`, `retryable=False`, `backend_name=None`
6. Implement `resolve_run_mode()` so it preserves `probe.reason_code` and applies only caller policy:
   - supported probe -> `mode="isolated"`
   - unsupported + `require_isolation=True` -> `mode="blocked"`
   - unsupported + `require_isolation=False` -> `mode="fallback"`
   - the fallback path uses `requires_acknowledgement=require_acknowledgement_for_fallback`
   - for unavailable isolation, set `message` to `Background runtime resolved to {mode} (reason: {reason_code}). {remediation}`
   - use this remediation mapping in the shared contract instead of ad hoc caller strings:
     - `xvfb_missing` -> `Install Xvfb to enable isolated background execution.`
     - `platform_unsupported` -> `Run without background isolation on this host until a supported isolated backend exists.`
     - any future unavailable reason -> include a non-empty remediation sentence before returning the decision
7. Implement `log_mode_resolution()` with stable prefixes:
   - every decision logs a single line beginning with `background runtime resolved:`
   - include `owner=`, `mode=`, `reason=`, `requires_ack=`, `task=`, and `message=` in that line
   - use `logger.info` for `isolated`, `logger.warning` for `fallback` and `blocked`
8. Implement `BackgroundRuntimeCoordinator` with `asyncio.Condition` and an async-context-manager `lease(run_metadata: dict[str, str], logger: logging.Logger | None = None)`:
   - keep `_active_run: dict[str, str] | None`
   - while busy, log a line beginning with `background runtime busy:` and include both waiting and active metadata keys `owner` and `task`
   - clear `_active_run` in `finally` and `notify_all()`
9. Expose a module-level singleton named `GLOBAL_BACKGROUND_RUNTIME_COORDINATOR`.

Update `opengui/backends/background.py` to consume the coordinator safely:
- Extend `BackgroundDesktopBackend.__init__()` with optional `runtime_coordinator` and `run_metadata` parameters.
- Default `runtime_coordinator` to `GLOBAL_BACKGROUND_RUNTIME_COORDINATOR`.
- In `preflight()`, acquire the lease before `display_manager.start()` and hold it until `shutdown()`.
- Make `preflight()` idempotent when `_display_info` already exists and `_stopped` is false, so the wrapper can survive the existing `GuiAgent.run()` preflight call without starting a second display.
- Keep the existing DISPLAY save/restore behavior intact.
- Release the lease in `shutdown()` after the display manager stops and DISPLAY is restored, even when `display_manager.stop()` raises.

Replace the Wave 0 runtime placeholders with real tests that assert the exact reason codes and log prefixes listed above.
