# Quick Task 260324-p41: Background GUI Trajectory Postprocessing

**One-liner:** Move GUI trajectory summarization and skill extraction off the synchronous `GuiSubagentTool.execute()` path so the main agent regains control immediately after the GUI agent returns.

## What Changed

- `nanobot/agent/tools/gui.py`
  Added `_schedule_trajectory_postprocessing()` and `_run_trajectory_postprocessing()` so trace summarization and skill extraction run in a background `asyncio` task after the GUI run completes.

- `nanobot/agent/tools/gui.py`
  Added `_background_postprocess_tasks` tracking plus `_wait_for_pending_postprocessing()` for explicit draining in tests or future shutdown flows.

- `nanobot/agent/tools/gui.py`
  Added `_handle_background_postprocess_done()` so background failures are logged and do not produce unhandled task warnings.

- `tests/test_opengui_p8_trajectory.py`, `tests/test_opengui_p3_nanobot.py`
  Updated existing assertions to explicitly wait for pending postprocessing when they validate summarizer/extractor side effects.

- `tests/test_opengui_p11_integration.py`
  Added regression coverage proving the GUI result returns before background postprocessing is released.

## Verification

- `uv run pytest tests/test_opengui_p8_trajectory.py tests/test_opengui_p3_nanobot.py tests/test_opengui_p11_integration.py`
- Result: `46 passed`

## Notes

- No user-facing payload shape changed.
- If the process exits immediately after a GUI run, background postprocessing may still be interrupted; this change optimizes main-agent latency rather than guaranteeing completion on abrupt shutdown.
