Create `opengui/planner.py` with:

1. **Dataclasses** (all frozen):
   - `AtomNode(instruction: str, node_id: str = "")` — leaf subgoal
   - `AndNode(children: tuple[SubgoalNode, ...], node_id: str = "")` — all children sequential
   - `OrNode(children: tuple[SubgoalNode, ...], node_id: str = "")` — try until one succeeds
   - `SubgoalNode = Union[AndNode, OrNode, AtomNode]` — type alias

2. **`_deserialize_tree(data: dict[str, Any]) -> SubgoalNode`** — recursive JSON-to-dataclass converter:
   - `{"type": "atom", "instruction": "...", "node_id": "..."}` -> AtomNode
   - `{"type": "and", "children": [...], "node_id": "..."}` -> AndNode (recurse children)
   - `{"type": "or", "children": [...], "node_id": "..."}` -> OrNode (recurse children)
   - Raise `ValueError` on unknown type

3. **`_create_plan_tool_schema() -> dict[str, Any]`** — returns the JSON Schema for the `create_plan` tool. The tool accepts a single `tree` parameter which is a recursive JSON structure with `type` ("and"/"or"/"atom"), optional `children` array, optional `instruction` string, and optional `node_id` string.

4. **`TaskPlanner`** class:
   - `__init__(self, llm: LLMProvider) -> None`
   - `async def plan(self, task: str, initial_screenshot: Path, memory_context: str) -> SubgoalNode`:
     - Build messages: system prompt (planning-specific, explaining AND/OR/ATOM semantics) + user message with task description + screenshot image
     - Call `self.llm.chat(messages, tools=[create_plan_tool_schema])`
     - Extract `create_plan` tool call from response
     - Deserialize `tree` argument via `_deserialize_tree()`
     - Return the SubgoalNode tree
     - Raise `ValueError` if no tool call, wrong tool name, or invalid tree structure
